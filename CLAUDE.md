# Claude Code Rules — traffic-sim

## RULE 1: Respond immediately with a plan before any work

When receiving ANY instruction — bug fix, feature, refactor, design task, anything — your FIRST response must be:
1. One-line confirmation of what was asked
2. A numbered task list with subtasks showing exactly what you will do

DO NOT open a file, run a command, or write any code until this plan is posted and visible to the user.

## RULE 2: Update the task list in chat as work progresses

After posting the initial plan, re-post or update the checklist each time a task or subtask completes.

- ✅ done
- ⏳ in progress
- ☐ pending

Keep it visible throughout. Silent execution is not acceptable.

## RULE 3: Progress updates during long-running tasks

Post a chat update every ~1 minute during long workflows. Update when switching tasks. Never go silent for more than a minute.
