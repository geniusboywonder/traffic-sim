---
name: Respond immediately with plan on instruction receipt
description: When receiving any instruction, immediately confirm it and outline the plan of tasks and subtasks before doing any work
type: feedback
---

When receiving an instruction, immediately respond to confirm you've understood it and outline your plan as a list of tasks and subtasks — before touching any code or files.

**Why:** User expects to see the plan upfront so they can correct course before work begins. Jumping straight into execution without confirming is frustrating.

**How to apply:** Every time an instruction arrives — whether a bug fix, feature, refactor, or design task — reply first with: (1) a one-line confirmation of what was asked, and (2) a numbered task list with subtasks showing exactly what you'll do.
